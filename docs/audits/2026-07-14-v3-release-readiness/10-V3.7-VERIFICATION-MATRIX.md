# SuperLocalMemory V3.7 verification matrix

This matrix defines the proof required to move every row in [09-V3.7-DEFECT-LEDGER.yaml](09-V3.7-DEFECT-LEDGER.yaml) to `verified`. It is deliberately stricter than the current default pytest selection.

## Proof hierarchy

| Level | Proof | What it can establish |
|---|---|---|
| L0 | Documentation assertion | Intended contract only; never closes a defect. |
| L1 | Static/code inspection | Reachability, wiring, and obvious mismatch; closes no behavioral row alone. |
| L2 | Isolated unit/property test | Local logic and invariants. |
| L3 | Multi-component contract test | Observable behavior across real components. |
| L4 | Built-artifact runtime test | Shipped wheel/sdist/npm/daemon/client behavior. |
| L5 | External/reproduction artifact | Independent runtime, benchmark, platform, or human audit evidence. |

P0 rows require L4 and, where practical, L5. P1 rows require at least L3 and L4 when package/client behavior is involved. P2 rows require the level specified by their acceptance criteria. Reading code is not verification.

## Evidence directory contract

Release-candidate evidence should be generated outside the source packages and published under a stable release bundle:

```text
release-evidence/v3.7.0/
├── manifest.json
├── source/
```

Large raw benchmark files may live in a release asset or immutable object store, but the manifest must carry a live URL, byte size, SHA-256, producer command, source commit, container digest, and creation time.

## Ingestion state assertions

Every entry point must expose the same state transitions:

| Stage | Durable before response? | Query behavior | Failure behavior |
|---|---|---|---|
| `raw` | Yes | Not silently presented as fully processed | Retryable pending operation exists |
| `queryable` | Yes for asynchronous success response | Searchable with `materialization_status=queryable` | Missing derived branches are explicit |
| `enriching` | Yes | Read-your-write allowed with partial-status trace | Stage checkpoint and error provenance retained |
| `complete` | Yes | All required declared indexes/derivations consistent | May not be set by duplicate shortcut |
| `failed` | Yes | Policy determines raw visibility; failure always visible to diagnostics | Idempotent retry or explicit terminal reason |

Tests must compare database state, index state, graph/temporal/provenance state, emitted events, and public response. Spying only on a mocked `engine.store()` is insufficient.

## Scope matrix

| Stored owner/scope | Querying profile | Expected |
|---|---|---|
| Same profile, personal | Same profile | Include |
| Other profile, personal | Current profile | Exclude |
| Same project, project | Different profile authorized for project | Include |
| Different project, project | Current profile | Exclude |
| Shared | Authorized peer/profile | Include |
| Shared | Unauthorized peer/profile | Exclude |
| Global | Same tenant/user contract permits global | Include |
| Global | Different tenant/user | Exclude |
| Deleted/forgotten | Any | Exclude from every index/cache/context/export |
| Archived | Any | Follow the one documented lifecycle contract |

The matrix runs independently through dense, lexical, graph, temporal, associative, fusion, rerank, cache, injection, export, and dashboard/API surfaces.

## Cross-client contract

Only a client with every row green may appear in the advertised integration count:

| Step | Assertion |
|---|---|
| Install | Existing user configuration is preserved and SLM ownership markers are bounded. |
| Identity | Client connects to the intended daemon/data root and rejects a neighbor. |
| Session | Safe, bounded, source-attributed context is injected. |
| Remember | Fact reaches canonical ingestion with caller/project attribution. |
| Same-client recall | Fact is recalled with trace and provenance. |
| Cross-client recall | Second client sees the fact only under authorized scope. |
| Update | New valid state supersedes old without destroying evidence history. |
| Forget | Every index/cache/context/export stops returning the fact. |
| Restart/offline | Client and daemon restarts do not lose or misattribute memory. |
| Upgrade | Supported upgrade retains behavior and identity. |
| Disable/uninstall | SLM-owned config is removed without deleting user memory or unrelated config. |

## Benchmark tracks

| Track | Required scope | Metrics |
|---|---|---|
| Historical paper ledger | All published SLM results | Exact release/commit, protocol, dataset scope, model, raw-artifact availability, limitations; never a V3.7 score |
| Local retrieval regression | Repository-owned deterministic and synthetic corpora; K=5/10/20 where supported | Recall-any/all, MRR, nDCG, precision, retrieved tokens, p50/p95/p99 |

| SLM Optimize | Declared content/provider corpus | Cache precision, stale rate, compression fidelity/ratio, cost, p95 |
| Laptop scale | Progressively increase corpus until a recorded hardware/time/resource limit; never extrapolate to an unrun size | Ingest throughput, quality-at-scale, p50/p95/p99, RAM, disk, compaction |
| Optional external research | LoCoMo, LongMemEval, and BEAM only when their complete declared protocol can be run later | Results are separately versioned research evidence and are not a V3.7 release gate |

No score target may be added after viewing test outputs. Protocol changes require a new manifest and invalidate comparison with earlier runs. V3.7 must state that current LoCoMo performance is unknown; historical paper scores may be cited only with their historical version and protocol.

## Security adversarial classes

- Stored direct and indirect prompt injection.
- Delimiter/markup breaking, Unicode confusables, encoded payloads, and oversized content.
- Caller identity spoofing and profile/project/tenant boundary attacks.
- Malicious local health/mesh/MCP processes.
- Symlink/path traversal and configuration ownership attacks.
- Cache key collision, cross-provider/tenant reuse, poisoning, and invalidation races.
- Migration tampering, corrupted state, rollback replay, and deleted-memory resurrection.
- Benchmark leakage through gold evidence, dataset summaries, question IDs, category labels, or failure-derived synonyms.

## Standard command families

Exact commands will be locked after test layout changes, but the release pipeline must include:

```bash
python -m pytest -q
python -m pytest -q -m "slow and not ollama"
python -m pytest -q -m native tests/vector
python -m pytest -q tests/test_benchmarks
python -m build
python -m twine check dist/*
npm pack --dry-run
```

These run inside test-owned homes/data roots. Built wheels, sdists, and npm tarballs are installed into clean environments and smoked independently. Source-tree success alone does not satisfy a package gate.

One person or model may hold multiple roles across the program, but not implementer and sole verifier for the same P0 row.

## Stable release condition

`3.7.0` is releasable only when:

1. Every YAML defect row is `verified`.
2. G0–G10 in the master plan are green.
3. Default and extended suites pass against built artifacts.
4. No test touches user state.
5. Registry staging artifacts match the frozen commit and evidence manifest.
6. Final rendered claims pass liveness and entailment review.
7. Known limitations contain no unresolved correctness, security, privacy, migration, recovery, benchmark-integrity, or public-contract defect.
