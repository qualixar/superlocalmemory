## Score Contract v2

V3.7 no longer presents retrieval ordering as answer probability.

| Field | Contract |
|---|---|
| `relevance_score` | Query-relative relevance, bounded to `0.0..1.0` |
| `ranking_score` | Internal diagnostic ranking utility; not a probability |
| `memory_confidence` | Confidence stored with the underlying assertion |
| `trust_score` | Evidence-policy trust signal |
| `answer_confidence` | `null` until a held-out calibrator is frozen and identified |

Canonical recall responses declare `score_contract_version: "2"`,
`calibration_status: "uncalibrated"`, `calibration_id: null`, and
`answer_confidence: null`. Legacy `score` and `confidence` aliases map to
`relevance_score` and `memory_confidence` for one compatibility release.

The current candidate composition is also stated narrowly: dense semantic,
BM25 lexical, temporal, Hopfield associative, and spreading activation can
produce candidates. Entity-graph information is a post-fusion score
enhancement in the current implementation, not a sixth candidate producer.
